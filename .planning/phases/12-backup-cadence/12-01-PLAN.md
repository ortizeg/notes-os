<objective>
Convert `BackingUpNotesRepository` from backing up the 92 MB `NoteStore.sqlite` before
EVERY write to backing it up ONCE per triage session ‚Äî before the session's first write
only. Over a 100-move session this is 1√ó copy instead of up to ~100√ó.

The before-every-write code path is removed entirely (no `backup_cadence` config hatch ‚Äî
RESOLVED 2026-06-08, SPEC ¬ß6 item 4). The `auto_backup_on_write: bool` master switch on
`BackupConfig` stays: when True ‚Üí back up once per session; when False ‚Üí never back up.
The BKUP-06 invariant is preserved for the session's first write ‚Äî a failed first-write
backup propagates `BackupError` and ABORTS that write (inner repo never called).

Purpose: eliminate per-move 92 MB churn while keeping the non-destructive restore
guarantee (BKUP-07, BKUP-08).
Output: per-session latch + `begin_session()` seam on `BackingUpNotesRepository`, a tiny
`BackupResettable` Protocol, session-start call sites in the CLI (`SortController.run`)
and TUI (`SortScreen._apply_inbox_refs`), updated threat-model docs, and an updated/extended
95%-coverage unit suite.
</objective>
