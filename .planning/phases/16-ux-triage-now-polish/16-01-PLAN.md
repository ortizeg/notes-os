<objective>
Guarantee that the triage **action line is never gated on body load**: from first paint the
category prompt (P/A/R/X/S) is live the instant the note title shows, even while the body is
still streaming and the preview pane shows the Phase-11 skeleton. Only `#note-preview` may show
a placeholder — `#prompt` never does (except during the refs-loading window, when input is also
inert, so there is no false affordance).

AUDIT RESULT (verified against the current `src/notes_os/screens/sort.py`): there is **no
production gap** — `_render_current_note` already renders `_CATEGORY_PROMPT` to `#prompt`
unconditionally at `AWAIT_CATEGORY` (outside the `if self._current_note is not None` body-cache
branch), and `on_key` is guarded only by `_loading` (refs not fetched), never by body load.
`_note_for_move()` builds a minimal `Note` from the ref so a move keystroke works before the
body lands. Therefore this phase is a **REGRESSION GUARD**, not a fix: it adds the Pilot test
that locks UX-04 plus a one-line invariant comment. No behavior change.

Purpose: close the test-coverage gap (the existing never-block test proves a *skip* mid-stream
but does NOT assert `#prompt` shows the live category prompt while the skeleton is on
`#note-preview`, nor that a *category move* keystroke is honored during body load).
</objective>
