<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backup dir → live Notes DB | restore() copies backup files back over the live notes_db_dir, replacing live state. |
| Caller → prune → filesystem | prune() deletes backup directories (shutil.rmtree) based on retention/older_than. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Data loss | restore() overwriting live DB while Notes is running could corrupt state | mitigate | restore() is a pure file op; the quit-Notes→restore→reopen procedure is documented in the docstring (and README later). restore refuses an incomplete backup (missing any of the 3 files → BackupError) so it never leaves a torn DB. |
| T-03-05 | Data loss | prune deleting the wrong / too many backups | mitigate | prune operates on list() (newest-first), keeps the N newest; retention guard documented; rmtree failures are best-effort logged (or raise) without silently losing track. SC4 test pins retention=3 of 5 → exactly 3. |
| T-03-06 | Tampering | integration test touching the real Notes DB | mitigate | Integration test uses TEMP source + backup dirs only, asserts notes_db_dir is under tmp_path, and skips on non-Darwin — the real ~/Library container is never read or written. |
| T-03-SC | Supply chain | new dependencies | accept | stdlib only (shutil, pathlib, datetime, logging). No new packages. |
</threat_model>
