<objective>
Build the backup foundation: the `BackupError` exception, the frozen `BackupConfig`
and `Backup` record models, the `BackupManager` (create + list of the three Notes DB
files), and the `BackingUpNotesRepository` decorator that fires a backup BEFORE every
write and aborts the write when the backup fails.

Purpose: Deliver BKUP-01 (auto-backup before every write), BKUP-02 (on-demand
timestamped + labelled backup), BKUP-03 (list backups), and BKUP-06 (a failed backup
raises BackupError and aborts the pending write). This plan makes Success Criteria 1
(auto-backup-before-move fires) and 5 (backup failure aborts the move) provable with
pure mocks — no real Notes DB.

Design decision — DECORATOR (Option A), chosen over injection:
`BackingUpNotesRepository` implements `NotesRepositoryProtocol`, wrapping an inner
`NotesRepositoryProtocol` plus a `BackupManager`. On `move_note` / `ensure_folder` it
calls `backup_manager.create()` first; if that raises `BackupError` it propagates WITHOUT
delegating (the inner repo's write is never reached). Read methods pass straight through.
Rationale: keeps Phase 2's `notes.py` UNTOUCHED, is trivially unit-testable with a
`MockNotesRepository` + mock `BackupManager`, and cleanly proves SC5 by asserting the
inner `move_note` was NOT called when the backup fails. (The PRD lists injection as the
literal wording but explicitly recommends the decorator — we take the recommendation.)

Module-layout decision: the two Pydantic models live in a dedicated
`src/notes_os/backup_models.py` so ONLY that module is added to the mypy
`disallow_any_explicit=false` override. `backup.py` defines no BaseModel and stays under
full explicit-Any checking.

Output:
- `src/notes_os/exceptions.py` — add `BackupError(NotesOSError)`.
- `src/notes_os/backup_models.py` — `BackupConfig` (frozen) + `Backup` (frozen record).
- `src/notes_os/backup.py` — `BackupManager.create/list` + `BackingUpNotesRepository`.
- `pyproject.toml` — add `notes_os.backup_models` to the override module list.
- `tests/test_backup_unit.py` — create/list/missing-source/decorator unit tests (tmp_path + mocks).

Restore() and prune() and the integration suite are plan 03-02 (depends_on this plan).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller → BackupManager → filesystem | A `label` string is interpolated into a backup directory name and a path is built under `backup_dir`. |
| Live Notes DB → backup dir | Three live SQLite files are copied; a partial/corrupt copy would compromise recovery. |
| Decorator → inner repo | The decorator gates every write behind a successful backup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | A malicious/typo'd `label` could traverse out of backup_dir (e.g. `../../etc`) | mitigate | Strip the label, reject path separators / os.sep; empty-after-strip → no label. Build the dir via `backup_dir / name` (pathlib), never string concatenation. |
