<interface_contract>
Locked signature (do NOT vary — Plan 10-02 calls it exactly this way):

    def get_inbox_note_bodies(self, offset: int, count: int) -> list[Note]

- `offset`: 0-based start index into the inbox notes in folder order.
- `count`: number of notes to fetch starting at `offset`.
- Returns: a list of fully-populated `Note` objects (id, title, body, preview),
  in folder order, id-aligned to the corresponding slice of `get_inbox_note_refs()`.
- An out-of-range or empty page (offset past the end, count == 0, empty inbox) returns `[]`.
- Add it to `NotesRepositoryProtocol`, `AppleScriptNotesRepository`,
  `BackingUpNotesRepository` (pass-through), and `MockNotesRepository`.

AppleScript range read (mirror `get_inbox_notes`, ONE osascript call):
- Source the same node set as `get_inbox_note_refs`: `set noteList to notes of inbox`.
- Read the contiguous slice `items (offset+1) thru (offset+count) of noteList` —
  AppleScript is 1-based, so a Python `offset` maps to AppleScript index `offset+1`,
  and the inclusive end is `offset+count` (clamp the end to the list length in the script
  so an over-long `count` does not error).
- Emit one record per note as `<id> _FIELD_SEP <name> _FIELD_SEP <body>`, records joined
  by `_RECORD_SEP` — identical wire format to `get_inbox_notes`, so the existing parse loop
  can be reused verbatim (id.strip(), title.strip(), body NOT stripped, preview via
  `_strip_html(body, self._config.preview_length)`).
- AppleScript-escape the inbox name exactly as the existing read methods do
  (`self._config.inbox_folder.replace('"', '""')`).
</interface_contract>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| osascript stdout → parser | AppleScript output (note ids, titles, HTML bodies) crosses into Python parsing; note text is attacker-influenced (a note body the user pasted from anywhere). |
| inbox_folder config → AppleScript source | The configured inbox name is interpolated into the AppleScript program string. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | get_inbox_note_bodies AppleScript builder | mitigate | Inbox name AppleScript-escaped via `.replace('"', '""')` exactly as existing read methods; unit test (8) asserts the doubled-quote appears in the generated script. |
| T-10-02 | Tampering | osascript stdout parser | mitigate | Reuse `_FIELD_SEP`(0x1F)/`_RECORD_SEP`(0x1E) control-char delimiters that cannot appear in Notes HTML/titles, so note text cannot forge a record boundary; malformed records (part count != 3) are logged and skipped, not trusted. |
| T-10-03 | Denial of Service | osascript range read on a huge inbox | accept | `count` is caller-controlled and bounded to a page (Plan 10-02 uses page size 200); the existing `_OSASCRIPT_TIMEOUT_SECONDS` ceiling still applies to this single call. Paging (not whole-inbox) is the design intent. |
| T-10-04 | Information Disclosure | note bodies in logs | mitigate | On a malformed record only the field COUNT is logged (`logger.warning("... got %d", len(parts))`), never the body — mirrors the existing read methods; no note content is logged. |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this plan (pure stdlib + existing deps); no Package Legitimacy Gate needed. |
</threat_model>
