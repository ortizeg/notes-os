<objective>
Wire extraction into the live sort flow: a Markdown task writer, an extraction UI
prompt (add-all / select-subset / skip), a config field for the output directory, and
the DI-injected controller integration that runs extraction ONLY after a successful
move AND ONLY when `task_extraction = true`. The headline guarantee is SC1: when the
flag is off (the default), the loop is byte-identical to Phase 4 — no extractor call,
no UI prompt, no file write.

Purpose: Delivers TASK-02 (user selection), TASK-03 (Markdown checkbox file), and
TASK-04 (off by default), and proves SC1 with an explicit test asserting zero
extractor/UI/writer calls when disabled.

Output:
- `TaskWriter` in extractor.py (append `- [ ] {task}` to `{dir}/YYYY-MM-DD.md`, injectable clock+dir).
- `extracted_tasks_dir` field on `FeaturesConfig` in config.py (default `~/.notes-os/extracted-tasks`).
- `prompt_task_selection` on `SortUIProtocol` + `RichSortUI`.
- `SortController` extended with injected extractor + writer, gated post-move extraction.
- Three test files: writer, UI, and controller-extraction (SC1 + enabled path).
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-SC | Tampering | pip installs | mitigate | No new dependencies (stdlib + existing pydantic/rich only); no package-manager install tasks — slopcheck N/A. |
