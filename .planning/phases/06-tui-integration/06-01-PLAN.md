<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keyboard → Textual screen | user keystrokes drive screen actions; only declared bindings act |
| app → AppleScript repo | only on the PRODUCTION (repo is None) path; tests inject a mock so no osascript runs |
| app → filesystem (backup_dir / tmp_path) | status reads BackupManager.list(); tests redirect to tmp_path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | NotesOSApp production wiring | mitigate | Production write path goes through BackingUpNotesRepository (backup-then-move) exactly as build_default_controller; not bypassed by the DI seam |
| T-06-02 | Information disclosure | HomeScreen backend label | mitigate | Backend label is the honest `sort-only (M1)` string; no fabricated LLM backend implying capabilities that do not exist |
| T-06-03 | Denial of service | blocking main().run() in CI | mitigate | Tests monkeypatch run() / use run_test(); main()'s real .run() never enters the event loop under pytest |
| T-06-SC | Tampering | pip install pytest-asyncio | mitigate | pytest-asyncio is a widely-used PyPA-adjacent test plugin (pypi.org/project/pytest-asyncio); pinned via pixi + pyproject; legitimacy low-risk, dev-only dependency |
</threat_model>
