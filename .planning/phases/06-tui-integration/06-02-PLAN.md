<objective>
Build SortScreen — the inbox triage flow as a Textual screen — and wire HomeScreen's "Sort Inbox" to it.

CRUX ARCHITECTURE DECISION (the whole reason the Router was built UI-agnostic): the Phase-4 `SortController.run()` is a SYNCHRONOUS readchar loop and Textual is async/event-driven. Do NOT call `SortController.run()` from a screen — it would block the event loop. Instead, SortScreen drives the UI-AGNOSTIC `Router` DIRECTLY via Textual key events: each keystroke calls `router.handle_category` / `handle_folder` / `handle_subfolder` / `handle_back`, and the screen renders the returned `RouteResult` and advances. The Router already calls `ensure_folder`+`move_note` on the injected repo; SortScreen records outcomes in a `SortSession` and writes the audit log at the end. The readchar `RichSortUI` + `SortController` + `python -m notes_os.sorter` CLI path is LEFT INTACT (parallel driver) — do NOT delete or modify it.

Purpose: TUI-03 (SortScreen runs the inbox triage flow end-to-end) and the TUI-05 nav on the sort screen. Makes SC2 provable.

Output: `src/notes_os/screens/sort.py`, HomeScreen wired to push it, app.py SCREENS registry entry, app.tcss additions, and a Pilot test proving one note routed end-to-end.
</objective>

  <action>
Create `src/notes_os/screens/sort.py` with `class SortScreen(Screen[None])`.

State the screen holds (instance attributes set in `__init__`/`on_mount`):
  - `_router: Router` built from `self.app.repo` + `self.app.app_config` (the backup-wrapped repo from 06-01's DI seam — so moves go through BackingUpNotesRepository; backup-then-move holds, SC2). Inject a fixed-year provider only in tests via the app; in production use the Router default.
  - `_session: SortSession` (fresh per screen entry).
  - `_notes: list[Note]` from `self.app.repo.get_inbox_notes()` (snapshot at `on_mount`).
  - `_index: int` pointer into `_notes`.
  - `_router_state: RouterState` and `_prev: RouteResult | None` carrying selection context between keystrokes (Router is stateless between calls — the SCREEN holds the context the controller used to hold on its call stack).

compose(): a header/title `Static` (`#note-title`), a body `Static` (`#note-preview`) rendering the current note's `.preview`, a `#prompt` Static showing the current options (PARA legend at AWAIT_CATEGORY; the numbered folder/subfolder list at AWAIT_FOLDER/AWAIT_SUBFOLDER), and a `#progress` Static (e.g. "Note 1 of 3"). On `on_mount`, if `_notes` is empty, render an "inbox empty" state and offer back-to-home.

Event handling — translate Textual key events into Router calls, mirroring controller.py's control flow but as discrete, non-blocking handlers (DO NOT loop/block):
  - Implement `on_key(self, event: events.Key)` (or per-key BINDINGS + actions) that dispatches based on `_router_state`:
      * AWAIT_CATEGORY: pass the key char to `self._router.handle_category(key, current_note)`. If `result.help_requested` → show the help overlay (the `?` legend) and stay. If `result.action == SKIP` → `_session.record_skip(note.id)` and advance to the next note. If `result.action == MOVE` → `_session.record_move(note.id, result.folder_path)` and advance (Router already did ensure_folder+move_note). If `result.state == AWAIT_FOLDER` → store `_prev=result`, set `_router_state=AWAIT_FOLDER`, render `result.options` as a numbered list. Unknown key → no-op re-render (ROUT-07).
      * AWAIT_FOLDER: map a numeric key (1-9) or a highlighted-list Enter to a 1-based index and call `self._router.handle_folder(index, _prev, note)`. MOVE → record + advance. AWAIT_SUBFOLDER → store `_prev`, render sub-options. Invalid index → re-render (no-op). Esc/B → `handle_back(AWAIT_FOLDER, _prev)` → back to AWAIT_CATEGORY.
      * AWAIT_SUBFOLDER: numeric/Enter → `handle_subfolder(index, _prev, note)` → MOVE → record + advance. Esc/B → `handle_back(AWAIT_SUBFOLDER, _prev)` → AWAIT_FOLDER (re-render folder options).
  - Wrap each MOVE attempt in `try/except NotesOSError` (mirror controller's T-04-10): on error `_session.record_error(note.id, str(exc))` and advance — one failure must not abort the screen. Import `NotesOSError` from `notes_os.exceptions`.
  - Advancing: increment `_index`, reset `_router_state=AWAIT_CATEGORY` and `_prev=None`, re-render the next note. When `_index` passes the last note, call `_finish()`.
  - `_finish()`: compute `summary = _session.summary()`, render it (counts moved/skipped/errors), call `_session.write_log(self.app.app_config.log_dir)`, and offer return to HomeScreen (`self.app.pop_screen()` or `push_screen('home')`).

TUI-05 nav on SortScreen (consistent with HomeScreen, declared as BINDINGS where natural):
  - Esc and B both = back one router level (subfolder→folder→category); at AWAIT_CATEGORY, Esc/B returns to HomeScreen (pop_screen) — but if a session is in progress (moves/skips recorded > 0), this is a "session in progress" boundary; route through the same quit-confirm seam (the actual confirm modal is 06-04 scope — here, popping to home is acceptable; leave a clearly named seam).
  - Q = quit the whole app (global binding from 06-01); since a sort session may be in progress, the confirm-on-quit modal is 06-04 — for now Q quits via the app binding.
  - ? = show the PARA key legend overlay for the sort context (a lightweight Static/notify of `[P]rojects [A]reas [R]esources [X] archive [S]kip  ↑↓/number select  Esc/B back`).

The TASK-EXTRACTION hook after a successful MOVE is 06-03's responsibility. Leave a single clearly-named seam method `_after_move(note)` called right after `_session.record_move(...)` that, in THIS plan, is a no-op (or logs). 06-03 fills it in (gated on `config.features.task_extraction`). Do NOT implement extraction here.

Add minimal styling to `src/notes_os/app.tcss` for `#note-title`, `#note-preview`, `#prompt`, `#progress` (surgical additions; do not restyle home).

| Boundary | Description |
|----------|-------------|
| keyboard → SortScreen | keystrokes drive Router transitions; unknown keys are no-ops (ROUT-07) |
| SortScreen → BackingUpNotesRepository | every move is preceded by a backup; failed backup aborts the write (BKUP-06) |
| Router → repo | ensure_folder always precedes move_note (router invariant) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering | move path bypassing backup | mitigate | SortScreen uses the app's BackingUpNotesRepository (DI seam); the SC2 test asserts backup fired before the move |
| T-06-05 | Denial of service | one bad note aborting the session | mitigate | each move wrapped in try/except NotesOSError → record_error + advance (mirrors T-04-10) |
| T-06-06 | Tampering | unrecognized keystroke causing invalid state | mitigate | Router returns no-op RouteResult for unknown keys / out-of-range indices; screen re-renders without advancing |
| T-06-07 | Denial of service | blocking SortController.run() in the event loop | mitigate | SortScreen drives the Router via discrete async event handlers — the blocking readchar controller is never invoked from Textual |
