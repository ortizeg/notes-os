## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user keystrokes -> controller | scripted/real input drives the whole sort loop |
| controller -> Apple Notes (via backing-up repo) | every move is preceded by a DB backup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Tampering | a write reaching Notes without a prior backup | mitigate | Controller wires AppleScriptNotesRepository inside BackingUpNotesRepository so a backup fires before every move/ensure_folder (backup-then-move, SC4); a failed backup aborts the write (BKUP-06 from Phase 3) |
| T-04-10 | Denial of Service | one note's error aborting the whole session | mitigate | Per-note writes are wrapped in try/except NotesOSError -> session.record_error and continue, so one failure does not lose the rest of the session (and is logged) |
| T-04-SC | Tampering | pip installs | accept | No new packages installed; no pyproject/entry-point edits in this plan (notes entry point deliberately NOT added) |
</threat_model>
