<objective>
Build the UI-AGNOSTIC PARA routing state machine in `src/notes_os/sorter/router.py`:
an explicit state enum + transition functions that consume the current state plus an
input event and return the next state and any action (resolve a FolderPath and call
`repo.move_note`, ensure the archive year folder, or skip). It performs NO terminal I/O.

Purpose: This is the decision core of the sort flow (ROUT-01..08). The UI (04-03)
captures keystrokes and renders; the router decides what they mean. Keeping it
I/O-free makes it deterministically testable to the 95% write-path-adjacent floor.

Output: `RouterState` enum, `Router` class (or transition functions), a `RouteResult`
describing the next state + resolved path/action, archive auto-year via an injected
clock; mypy override entry if a BaseModel is defined; full-transition unit tests.
</objective>
