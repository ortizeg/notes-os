<applescript_patterns>
The osascript output contract MUST be machine-parseable. Use a record-delimited,
field-delimited text format emitted by AppleScript, NOT AppleScript's default
list coercion (which is ambiguous to parse). Recommended approach:

- get_inbox_notes: iterate `notes of folder INBOX`, for each emit
  `id` + US (unit separator, char 31) + `name` + US + `body`, joined by
  RS (record separator, char 30) between notes. Set AppleScript text item
  delimiters explicitly. Empty inbox → empty string output. Parse in Python by
  splitting on RS then US; skip empty trailing records.
- get_para_structure: iterate the four configured PARA root folders; for each
  root, list its immediate subfolders by `name`. Emit
  `rootName` + US + `subfolderName` (one record per subfolder), and also emit a
  bare `rootName` record for roots with zero subfolders so the root still appears.
  Parse into `ParaStructure(roots=..., subfolders={root: [...]})`.

Encoding: pass scripts as a single string to `osascript -e`. Use unit/record
separators (chr(31)/chr(30)) so titles containing commas, quotes, apostrophes,
newlines, and Unicode survive parsing intact. Document the exact delimiter chars
in a module-level constant.
</applescript_patterns>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python → osascript (subprocess) | Folder/inbox names from BridgeConfig are interpolated into an AppleScript string and executed by a subprocess. |
| Apple Notes → Python (parsing) | Note titles/bodies are attacker-influenceable free text (the user's own notes) parsed by delimiter splitting. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | _run_osascript subprocess call | mitigate | Always invoke `osascript` with list args (`["osascript","-e",script]`), never `shell=True`; justified `# noqa: S603` only on this fixed trusted-binary call. No user string reaches a shell. |
| T-02-02 | Injection | AppleScript string built from config folder names | mitigate | Folder names come from frozen BridgeConfig (not arbitrary runtime input in this phase); AppleScript-escape embedded double-quotes when injecting names. Full untrusted-path injection hardening tracked when router supplies dynamic paths (02-02 move_note). |
| T-02-03 | Tampering | Delimiter parsing of note titles/bodies | mitigate | Use chr(31)/chr(30) unit/record separators (not commas/newlines) so note content containing punctuation, quotes, or newlines cannot forge record boundaries. |
| T-02-SC | Tampering | pip installs | accept | No new packages added — Pydantic already in pyproject from Phase 1; stdlib-only HTML stripping (html.parser). No package-legitimacy gate needed. |
</threat_model>
