<reference_consultation>
For the `move note` and `make new folder` AppleScript, consult `memo`
(https://github.com/antoniorodr/memo) for the osascript move/create patterns —
specifically how it addresses a note by `id`, resolves a destination folder, and
calls `move ... to folder ...`. Do NOT import or vendor memo. If no network:
use the standard Notes dictionary: `move note id "x:y:..." to folder "Name"`,
`make new folder with properties {name:"X"}`, `folder "X" of folder "Y"`,
and existence checks via `exists folder "X"`.
</reference_consultation>

<applescript_patterns>
Path resolution (FolderPath is a tuple of folder names, root-first):
- 2-level path `(root, folder)` → `folder FOLDER of folder ROOT`.
- 3-level path `(root, folder, subfolder)` → `folder SUB of folder FOLDER of folder ROOT`.
Build the nested `folder ... of folder ...` reference from the tuple in reverse.

move_note: AppleScript pattern — first verify the destination exists
(`if not (exists <folderref>) then error "FOLDER_NOT_FOUND"`), then verify the note
exists (`if not (exists note id NID) then error "NOTE_NOT_FOUND"`), then
`move note id NID to <folderref>`. Emit a sentinel token on success (e.g. `return "OK"`).
Map results in Python: osascript non-zero → NotesError(stderr); stderr/stdout containing
`FOLDER_NOT_FOUND` → FolderNotFoundError; `NOTE_NOT_FOUND` → NotesMoveError.
move_note MUST NOT call ensure_folder (caller responsibility — document this).

ensure_folder: idempotent create of a (possibly nested) path. For each prefix of the
path, `if not (exists <prefixref>) then make new folder ... ` so existing levels are
left untouched (no duplicate) and missing levels (e.g. Archive then 2026) are created
in order. A fully-existing path is a no-op. Non-zero osascript → NotesError.
</applescript_patterns>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Router → bridge → osascript | A resolved FolderPath (tuple of folder names) is interpolated into a `move`/`make new folder` AppleScript and executed. |
| osascript → Python | osascript stderr/stdout (incl. sentinel error tokens) is inspected to choose a typed exception. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Injection | _folder_reference builds AppleScript from FolderPath names | mitigate | AppleScript-escape embedded double-quotes in every folder name before interpolation; invoke via list-arg osascript (no shell). Folder names in this phase originate from discovered PARA structure / config, not free user text. |
| T-02-05 | Elevation/Tampering | move operation could move to an unintended folder | mitigate | Verify destination `exists` before `move`; raise FolderNotFoundError instead of silently creating (move_note never calls ensure_folder — separation of concerns prevents accidental folder creation on a typo'd path). |
| T-02-06 | Denial of Service | A failing osascript could crash the triage session | mitigate | All failures raise typed NotesError subclasses (NotesOSError descendants) so the caller (router/UI, later phases) catches and logs a warning and continues — verified by plan 02-03's failing-stub tests. |
| T-02-SC | Tampering | pip installs | accept | No new packages — stdlib + Pydantic (already vetted Phase 1). No legitimacy gate required. |
</threat_model>
