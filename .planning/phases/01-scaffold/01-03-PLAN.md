Create `scripts/setup-branch-protection.sh` — an idempotent, documented bash script. It MUST be
safe to run with no GitHub remote present:
- `#!/usr/bin/env bash` and `set -euo pipefail`.
- Detect the remote: `if ! git remote get-url origin >/dev/null 2>&1; then echo "No 'origin'
  remote configured — skipping branch protection. Add a GitHub remote and re-run." >&2; exit 0; fi`.
- Detect gh: `if ! command -v gh >/dev/null 2>&1; then echo "gh CLI not installed — skipping." >&2;
  exit 0; fi`. Also verify auth (`gh auth status`) and skip gracefully if not authenticated.
- Resolve `OWNER/REPO` from the origin URL (or `gh repo view --json nameWithOwner`).
- Apply repo merge settings: `gh repo edit "$REPO" --enable-squash-merge --enable-merge-commit=false
  --enable-rebase-merge=false --delete-branch-on-merge` (use the github-repo-setup skill's exact
  flags).
- Apply branch protection on `main` via `gh api` (PUT /repos/{owner}/{repo}/branches/main/protection)
  requiring: a passing PR, required status checks (the CI job names from 01-02: lint, typecheck,
  test — list them so merges block until green), required PR review, and
  `required_pull_request_reviews` with CODEOWNERS enforcement (`require_code_owner_reviews: true`).
  Reference the github-repo-setup skill for the exact JSON payload.
- Echo a success summary at the end. NOTE in a comment block at the top: "Run this once after the
  GitHub remote is created. It is idempotent — safe to re-run."

Mark the script executable (`chmod +x scripts/setup-branch-protection.sh`). The script lives under
`scripts/**` which the ruff per-file-ignore allows T20 for (echo is shell, not python — no ruff
concern, but document that scripts/ is the allowed home for tooling).

Do NOT execute the gh protection call as a hard plan step — if a remote happens to exist and gh is
authenticated, running the script is fine and encouraged; otherwise just verify it exits 0
gracefully.
