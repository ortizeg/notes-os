## Goal

Make inbox triage feel instant: no Apple Event ever blocks a keystroke, on skip
*or* move — while preserving the non-destructive backup guarantee.

## v3.0 Requirements

### Performance (PERF)
- [ ] **PERF-01**: On a typical inbox (≤250 notes), every note's preview is warm within ~0.5 s of opening Sort — no per-skip "Loading" wait. → Phase 10
- [ ] **PERF-02**: On a large inbox (>250 notes), previews load in background pages without blocking any keystroke, with a live, non-blocking `Loading previews… N/M` indicator that clears on completion. → Phase 10
- [ ] **PERF-03**: A note reached before its page has loaded (skipping faster than the load) still resolves its own preview via a single-note fallback fetch. → Phase 10
- [ ] **PERF-04**: A move shows instant confirmation and advances to the next note without freezing the UI; the backup + move run off the event loop. → Phase 13
- [ ] **PERF-05**: A failed move/backup is surfaced to the user and the note is retained for retry — never silently dropped, and one failure never aborts the session. → Phase 13

### Experience (UX)
- [ ] **UX-01**: A pending preview displays a dim skeleton placeholder rather than literal "Loading preview…" text. → Phase 11
- [ ] **UX-02**: User can undo (`U`) the most recent action — restoring a skipped or moved note to its prior position (and original folder for moves), with session counts corrected; `U` is repeatable, walking back unbounded within the session. → Phase 14
- [ ] **UX-03**: User can resume an interrupted session at the correct note (matched by note id) or choose to start over; stale saved state (inbox changed) is ignored safely. → Phase 15
- [ ] **UX-04**: From first paint, category keystrokes (P/A/R/X/S) are accepted while previews still stream; the action/prompt line never appears blocked. → Phase 16

### Backup (BKUP)
- [ ] **BKUP-07**: A restore point is captured before the *first* write of each session (replacing before-every-write), preserving rollback while eliminating per-move churn. → Phase 12
- [ ] **BKUP-08**: The before-every-write backup path is removed entirely — per-session is the sole cadence (no config hatch); a failed first-write backup still aborts that write (BKUP-06 invariant preserved for the session's first write). → Phase 12

## Out of scope (this milestone)
- Persistent osascript / JXA bridge replacement — deferred; bulk loading makes call-count low enough that it isn't worth the complexity yet.
- Editing note contents from the TUI.
- M2 Distillation Engine work (separate milestone, worktree `cool-crane`).
